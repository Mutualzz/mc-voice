# Contributing

PRs welcome. Redistributing unofficial builds is still not allowed — see [`LICENSE`](./LICENSE).

## Credit

- Your git author stays on landed commits
- Merged PRs keep GitHub attribution
- Notable changes may appear in release notes

## How to contribute

1. Fork and branch.
2. Keep the change focused.
3. Test with `./gradlew buildAll` (or e.g. `./gradlew :1.21.1-fabric:remapJar`).
4. Open a PR describing what changed and why.

By contributing, you grant Mutualzz a license to use your work under the project license. You do **not** get rights to redistribute Mutualzz builds.
